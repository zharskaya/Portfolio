## How I Work

I don't start with screens.

I start with:
- what's actually broken
- what users believe is happening
- where the system lies or hides too much

Then I:
- reframe the problem
- test mental models early (prototypes > specs)
- design for clarity, not just completion
- build systems that scale, not one-offs
